| Dimension        | Small (1–4 IT staff)                | Standard (5–15 IT staff)   | Large (15+ IT staff)               |
| ---------------- | ----------------------------------- | -------------------------- | ---------------------------------- |
| Timeline         | 18–24 months                        | 12 months                  | 9–12 months (compressed)           |
| Phases           | 3–4 sequential                      | 6 overlapping              | 6 overlapping + parallel tracks    |

## Small agency path — month-by-month

For agencies with very small IT teams the full 6-phase, 12-month plan is not realistic. The Small path focuses on what delivers value fastest and treats each milestone as a complete, defensible deliverable on its own.

- **Months 1–3 — Governance sprint + AI foundations.** Hire external counsel for a 2-day governance intensive ($3–5K) that produces the AUP, risk matrix, and committee charter. Run Track 1 for all staff in parallel. This alone is a complete outcome that satisfies most legislative AI policy requirements.
- **Months 4–6 — Champions network + vendor evaluation.** Launch monthly champions meetups. Instead of building infrastructure from scratch, evaluate managed AI services (Azure AI Services, AWS Bedrock, Google Vertex) using the vendor evaluation framework in the guide.
- **Months 7–12 — Managed platform + simplified starter project.** Use cloud-managed services instead of custom modules. Build 1–3 lightweight modules (Auth, API wrapper, AI orchestration) on top of managed infrastructure. Deploy one starter project using pre-built frameworks (LangChain, Semantic Kernel, etc.).
- **Months 13–18 — Iterate and expand.** With the first app in production, iterate based on real usage. Add modules as needed. Consider hiring 1–2 contract engineers for month 13+ to build out the platform further.

> **Phase B checkpoint (Month 9).** At the end of Phase B, small agencies should evaluate whether they have the capacity and budget to continue to Phase C. If not, the governance framework, training, and 3-module foundation already delivered remain valid on their own; the agency can resume at Phase C when resources allow without redoing earlier work.

> **Off-Ramp: Governance only.** Agencies that complete only the Governance Sprint (Months 1–3) still walk away with an adopted AI policy, risk classification, and review committee. That is a complete, standalone deliverable that satisfies legislative requirements and positions the agency for future phases when budget and staffing allow.

## Budget by phase

Year-1 ranges, with the Small column showing what's achievable using managed services and external counsel instead of in-house build-out.

| Phase              | Primary costs                                | Standard   | Small               |
| ------------------ | -------------------------------------------- | ---------- | ------------------- |
| 1. Governance      | Staff time, external legal review            | $5K–$25K   | $3K–$8K (sprint)    |
| 2. Education       | Facilitator time, materials, optional LMS    | $10K–$50K  | $5K–$15K            |
| 3. Infrastructure  | Cloud compute, licenses, security tools      | $30K–$100K | $10K–$30K (managed) |
| 4. Dev Stack       | Staff time, tooling licenses (IDE, AI tools) | $15K–$40K  | $5K–$15K            |
| 5. Platform        | Developer time (largest cost), testing infra | $50K–$150K | $20K–$50K (pre-built) |
| 6. Starter Project | Dev time, user testing, deployment           | $30K–$80K  | $15K–$40K           |

> **Procurement reality check.** Government procurement cycles typically require 12–18 months of lead time. Agencies reading this today are likely budgeting for FY2027 or FY2028. Use the free/low-cost Phase 1 governance resources immediately while you position for budget requests.
