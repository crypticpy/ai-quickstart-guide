# Slide 1: What AI Actually Is

Main point: AI is a practical workplace tool category, not a magic box or a single product.

Bullets:

- Today is shared vocabulary, not technical certification
- We will separate useful tools from hype

# Slide 3: AI In One Sentence

Main point: AI is a broad term for systems that perform tasks associated with human intelligence.

Bullets:

- It can classify, predict, search, summarize, draft, or generate
- It is not one product or one vendor
- It works differently depending on the tool and task

Speaker notes:

Keep this definition simple. Staff do not need a history of AI. The important point is that AI is a category, not a single app. Some AI tools are old and quiet, like spam filters or routing tools. Others are newer and more visible, like chatbots and image generators.

Image guidance:

Simple umbrella diagram labeled "AI" with branches for search, summarize, draft, classify, and generate.

Evidence and review notes:

No external claims.

# Slide 4: Machine Learning In One Sentence

Main point: Machine learning is a type of AI where systems learn patterns from examples instead of being explicitly programmed for every case.

Bullets:

- The system learns from training examples
- It finds patterns that can support predictions or classifications
- It can fail when the examples are incomplete, biased, stale, or mismatched

Speaker notes:

Use a familiar example if helpful: sorting emails into spam and not spam, or classifying service requests by category. Emphasize that "learns" does not mean the system understands the way a person does.

Image guidance:

Simple flow: examples in, pattern learned, new item classified. Avoid brain imagery if it implies human-like understanding.

Evidence and review notes:

No external claims.

# Slide 5: Generative AI In One Sentence

Main point: Generative AI creates new text, images, code, audio, or other content based on patterns it learned from data.

Bullets:

- It produces new content instead of only scoring or sorting
- Chatbots and writing assistants are common examples
- It can sound confident even when it is wrong

Speaker notes:

Most "AI" in current workplace conversations means generative AI. This is the category staff are most likely to encounter through tools like chatbots, writing assistants, code assistants, or document summarizers.

# Slide 8: Category 3 - Search And Retrieval

Main point: Retrieval tools help answer questions from a bounded set of documents.

# Slide 9: What Is Underneath A Chatbot

Main point: A large language model predicts plausible next words; it does not know whether a statement is true.

Bullets:

- It learned patterns from large bodies of text
- It predicts likely next pieces of language
- It can produce polished wrong answers
- Confidence is not verification

Speaker notes:

This is the most important concept in the session. Every misuse traces back to forgetting that fluent text is not the same as verified truth. Tell the room that the demo will intentionally show this failure mode.

Image guidance:

Plain process diagram: prompt, pattern prediction, draft response, human verification. Avoid human brain or sentient-machine imagery.

Evidence and review notes:

No external claims.

# Slide 10: Two Things AI Cannot Reliably Do

Main point: AI cannot reliably verify truth or replace the context and accountability of agency staff.

Bullets:

- It cannot tell you whether something is true
- It cannot know local context unless supplied and verified
- It cannot own public responsibility for a decision
- Staff remain accountable for final work

Speaker notes:

Say this plainly. This is not anti-AI; it is how safe AI use works. AI can help draft, search, summarize, or organize. It cannot be the accountable public servant.

Image guidance:

Two-column slide: "AI can help" and "People remain accountable." Use simple icons, not decorative imagery.

Evidence and review notes:

No external claims.

# Slide 11: Live Demo - What To Watch For

Main point: The demo is about learning how outputs behave, not making the tool look impressive.

Bullets:

- Watch how the prompt changes the output
- Check whether the answer matches the source
- Notice when the tool sounds confident
- Ask what a human still needs to review

Speaker notes:

Use a one-page public document, such as public board minutes, a published report, or a policy excerpt. Do not use staff PII, case files, resident records, protected health information, or confidential documents.

Image guidance:

Simple "observe, compare, verify" process graphic.

Evidence and review notes:

Confirm demo content is public or approved for training use.
