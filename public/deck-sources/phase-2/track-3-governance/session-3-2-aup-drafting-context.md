# Deck Source: Track 3 Session 3.2 - AUP Drafting Context

## How to use this file

Use this as a short framing deck before drafting the AI Acceptable Use Policy. Localize policy owners, review path, and approved tools.

## Deck instructions

- Audience: governance, legal, HR, security, privacy, records, and program designees
- Session length: 90 minutes
- Deck length: 7 slides
- Purpose: context only; most time is drafting
- Do not invent policy authority, legal requirements, labor commitments, or approved tools

## Import prompt

Create a short context-setting deck. Follow slide breaks exactly. Use speaker notes as presenter notes.

---

# Slide 1: Drafting The AI Acceptable Use Policy

Main point: The AUP tells staff what they can do, what they cannot do, and where to ask.

Bullets:

- Approved use
- Prohibited use
- Data handling
- Human review
- Reporting and escalation

Speaker notes:

Open by saying the AUP is staff-facing. It should be clear enough that non-specialists can follow it.

Image guidance:

Policy document with plain-language checklist.

Evidence and review notes:

Route through local legal, HR, labor, security, records, and accessibility review as applicable.

---

# Slide 2: The Policy Job

Main point: The AUP should reduce ambiguity without freezing every future tool choice.

Bullets:

- Rules for staff behavior
- Reference to Approved AI Tools List
- Data class limits
- Review requirements
- Exception path

Speaker notes:

Keep tool lists out of the policy body when possible. A maintained tools list is easier to update.

Image guidance:

Policy core connected to maintained tools list.

Evidence and review notes:

Confirm local document-control practice.

---

# Slide 3: Staff Need Clear Boundaries

Main point: The AUP should answer common staff questions directly.

Bullets:

- Which tools can I use?
- What data can I enter?
- When must a human review?
- What should I do if output is wrong?
- How do I report a concern?

Speaker notes:

If the policy cannot answer these questions, staff will improvise.

Image guidance:

FAQ-style slide.

Evidence and review notes:

No external claims.

# Slide 4: Data Handling Rules

Main point: Data rules should be concrete and conservative.

Bullets:

- Public data
- Internal data
- Confidential data
- Regulated or sensitive data
- Unknown or mixed data

Speaker notes:

Use the agency's existing data classification if one exists. Do not create a parallel classification system unless necessary.

Image guidance:

Data classification ladder.

Evidence and review notes:

Align with local data classification and records policy.

# Slide 5: Human Review Rules

Main point: The AUP should name where AI output cannot be used without review.

Bullets:

- Public communications
- Legal or policy interpretation
- Rights, benefits, safety, money, employment
- Procurement or contract language
- Personnel or disciplinary matters

Speaker notes:

Local counsel and HR should adjust this list. The goal is clarity for staff.

Image guidance:

Review-required checklist.

Evidence and review notes:

Confirm with legal and HR.
