# Deck Source: Track 3 Session 3.1 - Risk Tiering Context

## How to use this file

Use this as a short context deck before the working session. Paste or upload into an AI presentation tool, then localize examples and legal references before use.

## Deck instructions

- Audience: AI Review Committee members and governance designees
- Session length: 90 minutes
- Deck length: 7 slides
- Purpose: orient the room for 10-15 minutes, then move into artifact work
- Do not invent legal requirements, tier definitions, or agency facts

## Import prompt

Create a short context-setting deck. Follow slide breaks exactly. Use speaker notes as presenter notes. Keep it brief so the group can move into working time.

# Slide 2: Why Tiering Matters

Main point: Tiering keeps low-risk ideas moving while giving higher-risk uses the review they deserve.

Bullets:

- Avoids one-size-fits-all review
- Helps staff know when to ask
- Gives vendors clearer expectations
- Creates a decision record

Speaker notes:

The committee should be able to say yes with conditions, not only no.

Image guidance:

Fast lane and deeper review lane.

Evidence and review notes:

No external claims.

# Slide 3: Baseline Tier Pattern

Main point: Start with three practical tiers and refine with agency-specific triggers.

Bullets:

- Tier 1: low-risk productivity support
- Tier 2: operational impact or sensitive context
- Tier 3: rights, benefits, safety, employment, liberty, or money

Speaker notes:

Use local names if different. The purpose is to create shared judgment.
