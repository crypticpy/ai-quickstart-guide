# Deck Source: Track 3 Session 3.4 - Procurement Guardrails Context

## How to use this file

Use this as a short framing deck before adapting AI procurement guardrails and questionnaire language.

## Deck instructions

- Audience: procurement, legal, security, privacy, program lead, and AI reviewers
- Session length: 90 minutes
- Deck length: 7 slides
- Purpose: frame decisions, then draft
- Do not invent vendor claims, legal requirements, contract authority, or pricing

## Import prompt

Create a short context-setting deck. Follow slide breaks exactly. Use speaker notes as presenter notes.

Bullets:

- Data rights
- Model use and substitution
- Security and logs
- Testing and bias review
- Exit and deletion

# Slide 2: Why Standard Terms May Be Insufficient

Main point: AI vendors may introduce risks not covered clearly in older templates.

Bullets:

- Data used to train or improve models
- Model provider subcontractors
- Changing model versions
- Opaque evaluation claims
- Weak export or deletion terms

Speaker notes:

Do not imply every vendor is unsafe. The point is that the agency needs to ask consistently.

Image guidance:

Contract with AI-specific callouts.

Evidence and review notes:

No external claims.

# Slide 3: Minimum Vendor Questions

Main point: Vendors should answer basic AI, data, security, and evaluation questions before pilot or procurement.

Bullets:

- What models and providers are used?
- Is agency data used for training?
- Where is data processed and stored?
- What logs can we export?
- How is accuracy measured?

Speaker notes:

These questions should apply even to no-cost pilots when agency data is involved.

Vendor questionnaire preview.

Evidence and review notes:

Align with local procurement rules.

# Slide 4: High-Impact Use Cases Need More

Main point: Higher-risk procurements need stronger testing, audit, and oversight terms.

- Bias or disparate-impact testing
- Audit access
- Human review requirements
- Incident notification
- Performance reporting

Speaker notes:
