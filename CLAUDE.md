# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Source of truth

This Astro Starlight site is the published form of three long-form planning documents in the parent directory:

- `../AI_Quickstart_PRD.md` — what content must exist; persona definitions; agency-size variants; deliverables list
- `../AI_Quickstart_Curriculum_Map.md` — Track 1–7 sessions, ADKAR/Kirkpatrick mapping, lab inventories
- `../AI_Quickstart_Gantt_Dependencies.md` — sequencing, hard/soft dependencies, 14 milestone gates (G-01–G-14), 5 human-side gates (H-01–H-05), Small Team 18-month variant (Appendix A)

Cross-cutting changes (e.g., adjusting Phase scope) usually need updates in all three docs and the relevant phase index here. The parent repo's `CLAUDE.md` covers the planning docs themselves.

## Common commands

```bash
pnpm install            # or npm install / yarn install
pnpm dev                # starts dev server at http://localhost:4321/ai-quickstart-guide/
pnpm build              # static build to dist/ (~99 pages, ~6s)
pnpm preview            # serve the built dist/
pnpm lint:links         # lychee link checker against dist/**/*.html (run after build)
```

There is no test suite. The build itself is the test: it surfaces YAML frontmatter errors, MDX parse errors, broken internal links (Starlight warns), and missing referenced pages. Always run `pnpm build` after content changes; clean output = ~99 pages built and 0 broken-link warnings.

## High-level architecture

**Astro 6 + Starlight 0.38 + Preact + astro-mermaid.** Single static site, no SSR, no API routes. Output is plain HTML deployed to GitHub Pages via `.github/workflows/deploy.yml`. Pagefind builds the search index at build time. `export-playbook.yml` produces a combined PDF/DOCX on each release tag via pandoc.

**Three things in `astro.config.mjs` matter and are not obvious:**

1. **`astro-mermaid` integration must come before `starlight`** in the integrations array — its remark plugin needs to run before Starlight processes the markdown. Reordering breaks Mermaid rendering.
2. **`Search` slot is overridden** with `./src/components/HeaderWithPicker.astro`. That wrapper renders Starlight's default Search verbatim plus the agency-size dropdown beside it. Touching the Search component requires re-rendering this wrapper.
3. **`BASE` is `/ai-quickstart-guide`** by default (configurable via `BASE_PATH` env var). All internal links in markdown should be written as bare paths (`/phase-1-governance/...`) — see remark plugin below.

### `src/plugins/remark-base-links.mjs`

Auto-prefixes any markdown link that starts with `/` (and isn't already prefixed) with the configured `BASE`. **Authors should always write bare paths** (`/phase-3-infrastructure/cicd-pipeline/`) — never hardcode `/ai-quickstart-guide/...`. The plugin keeps things working when forks deploy under a different base path. (A few legacy hero links in `src/content/docs/index.mdx` still use the prefixed form; these work but are non-canonical.)

### Agency-size variant system

The Small/Standard/Large variant selector is a custom integration on top of Starlight's `<Tabs syncKey="...">` primitive. Touching any of these means understanding the contract:

- **`src/lib/agencySize.ts`** owns the storage key (`starlight-synced-tabs__agency-size` — exact format Starlight expects) and a custom event (`aqg:agency-size-change`) for same-tab live sync. `setAgencySize()` writes localStorage AND walks `starlight-tabs[data-sync-key="agency-size"]` elements to programmatically click the matching tab; storage events only fire across tabs, so this is required for the picker and dropdown to live-update visible Tabs.
- **`src/components/AgencyPathPicker.tsx`** (Preact, hero on home + pick-your-path) and **`src/components/AgencyPathDropdown.astro`** (in the header via `HeaderWithPicker.astro`) both read/write through this lib. They stay in sync via the custom event.
- **Every phase index uses `<Tabs syncKey="agency-size">`** with TabItems labelled exactly `"Small"`, `"Standard"`, `"Large"`. **Tab labels must be these exact strings** — `setAgencySize` matches by `tab.textContent.trim() === size`. Don't write `"Small (1–4)"` etc.; the dropdown can show longer labels because select uses `value` separately from displayed text.

### Content layout

`src/content/docs/` mirrors the URL structure. Phase indexes live at `phase-N-<slug>/index.mdx`. Sidebars are largely autogenerated per directory in `astro.config.mjs`; `sidebar.order` in frontmatter controls ordering inside a directory. The Start Here sidebar is hand-curated.

Interactive components live in `src/components/` as `.tsx` (Preact) or `.astro`. They're embedded in MDX with `<Component client:load />` (the `client:load` is required — these are interactive). New shared state belongs in `src/lib/` next to `agencySize.ts` and `printDocument.ts`.

## Authoring conventions worth not relearning

**Phase index template (Phases 3–6 set the standard, Phases 1–2 now follow):** Objective → "How this phase scales for your agency" Tabs block → Deliverables → Sequencing within Phase X → Hard dependencies → What Phase X does NOT cover → `## Off-Ramp — <Name>` → Plain-English Guide to Phase X Terms → Research basis → Related. Adding a phase index? Use `phase-3-infrastructure/index.mdx` as the reference shape.

**Required callouts on phase indexes:** `## Plain-English Guide to Phase X Terms` whenever a phase introduces ≥3 jargon terms. `## Research basis` paragraph citing versioned sources (NIST SP 800-204D, OMB M-24-10, etc.) — not generic "industry best practices."

**Off-Ramp callouts** (`## Off-Ramp — <name>`) mark complete-and-valuable stop points. The five named off-ramps are: Governance Only (Phase 1), Tracks 1+3+7+intake+commitment+pulse (Phase 2), Sandbox-only (Phase 3), Stack-Conformant Pilots (Phase 4), Platform Without Starter Project (Phase 5), Pilot Without Production Launch (Phase 6). Don't invent new ones casually.

**MDX gotchas:**

- **Frontmatter titles containing a colon must be quoted:** `title: "Lab 3 — RAG: Retrieval-Augmented Generation"`. Unquoted colons break YAML parsing.
- **`</TabItem>` indented inside a bulleted list** is parsed as continuation of the list. Keep TabItem tags flush-left.
- **`<1 hour`** is parsed as a JSX tag start. Write `under 1 hour` instead.

**File and directory naming:** kebab-case, no exceptions. Phase directories prefixed with phase number (`phase-3-infrastructure/`).

## Deferred from upstream design

The original Gantt/PRD specified MkDocs Material + `mike` versioning + `mkdocs-pdf-export-plugin`. This codebase is the v1.2 revision — Astro Starlight + git tags + pandoc. The Gantt doc's Section 4–5 reflects what's actually shipped. If a planning doc and this site disagree, this site is the canonical implementation.
