# Code of Conduct

## Our Standard

This project welcomes contributors from government, civic tech, academia, industry, and the public. We expect participation to be professional, respectful, and focused on improving the guide.

Examples of expected behavior:

- Use welcoming and inclusive language.
- Assume good intent while still reviewing carefully.
- Critique ideas, content, and code rather than people.
- Be clear when something is opinion, local practice, or sourced fact.
- Respect that agencies vary widely in size, staffing, legal context, and technical maturity.

Examples of unacceptable behavior:

- Harassment, discrimination, insults, or personal attacks.
- Publishing private information without explicit permission.
- Submitting real resident, staff, credential, procurement, or security data.
- Bad-faith edits, spam, or vendor promotion disguised as guidance.
- Repeatedly ignoring maintainer direction on scope, tone, sourcing, or safety.

## Enforcement

Maintainers may edit, reject, or close contributions that violate this code. Serious or repeated violations may result in blocked participation.

To report an issue, open a private security or conduct contact path if one is available for the hosting organization. If not, open a GitHub issue with only the minimum public detail needed and ask maintainers to move the discussion to a private channel.

## Attribution

This code of conduct is adapted from common open-source community standards and tailored for a public-sector documentation project.
